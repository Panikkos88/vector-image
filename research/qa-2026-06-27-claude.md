## ⚠️ Warnings
- **W1 — Dev/hidden routes live in production.** `?engine=`, `?paletteK=`, `?paletteOptimize=`,
  `?asset=` all work on the public Cloud URL (`app.js:6334-6337`, `4179-4180`). Same-origin only
  (not a security hole) but lets any visitor bypass the auto-router / load internal benchmark
  assets. GATE behind localhost or a build flag before any public launch.
- **W2 — Optimizer runtime + unverified WIP.** `optimizeRegionTrace` / `optimizePaletteTrace`
  run many full-res trace+measure candidates (~4-7s/trace). The region downscale-eval / promote=6
  path was never browser-verified (OPEN in Next Steps #4). Verify or remove.
- **W3 — Fragile Palette↔Region fill coupling.** The Palette engine reuses `traceRegionsToSvg`
  and implicitly relies on `fitRegionAdaptive`'s flat threshold (14) choosing flat. This is why
  fine-text gradient-bloated (fixed via k=3) and why dark-glow bands the glow. An explicit
  flat-only mode for the palette path would decouple the two engines' tuning and prevent
  regressions.

## 📝 Notes
- **N1** — Init `catch(e){/* ignore */}` blocks (`~6333-6343`) silently swallow dev-route +
  `loadQueryAsset` errors; a real failure would be invisible. Consider a dev `console.warn`.
- **N2** — `fitRegionAdaptive` flat threshold **14** is the dark-glow lever (bg SSE 13.2 just
  under). Do NOT naively lower it — shared with the Region engine; would re-bloat fine-text.
  Tune per-engine if addressing glow.
- **N3** — Stale engines (`coverage`, `vtracer`, `imagetracer`, `experimental`) still reachable
  via `?engine=` but not part of the product (router only picks palette/regions). Harmless
  dead-weight; consider trimming the dev allowlist or documenting them as comparison-only.
- **N4** — 32 `*.bak-*` snapshots in `app/` (gitignored). Working-dir clutter; prune periodically.
- **N5** — Single 6,343-line `app.js`, two agents, no modules. No dup defs today, but
  merge/maintainability risk grows; consider splitting engines into files eventually.
